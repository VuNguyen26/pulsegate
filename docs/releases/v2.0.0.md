# PulseGate v2.0.0

Status: Release preparation in progress; final validation and annotated tag pending.

PulseGate v2.0.0 is the Product/Platform v2 repository and product-documentation milestone for Sprints 61-80.

## Release Scope

The release packages the already implemented Product/Platform v2 scope:

- Admin Dashboard and fixed GET-only server-side BFF resources.
- Public, static-first Developer Portal.
- Host-based and weighted upstream routing.
- Configured service discovery with bounded health-aware failover.
- Kubernetes manifests and local render/runtime documentation.
- Manual OpenTelemetry tracing foundations.
- Structured JSON logging with Loki, Grafana Alloy, and Grafana integration.
- Admin RBAC and credential-boundary hardening.
- Shared UI loading, empty, error, accessibility, and responsive polish.
- Bounded end-to-end demonstration and lightweight k6 validation.
- Documentation, runbook, architecture, requirements, and navigation cleanup.

## Release Validation Contract

Before the release is marked complete:

- Run full workspace tests, typecheck, and production builds.
- Run `npm.cmd run validate:release`.
- Validate Docker Compose configuration and the ten-service inventory.
- Render all existing Kubernetes Kustomize targets without applying them.
- Run the existing bounded GET-only runtime demonstration.
- Run the existing bounded k6 smoke.
- Record bounded database deltas and runtime cleanup evidence.
- Confirm private npm workspace versions remain `0.1.0`.
- Confirm `package-lock.json` is unchanged.
- Confirm protected annotated tag `v1.0.0` is unchanged.
- Confirm the repository is clean and synchronized with `origin/main`.

## Version and Tag Convention

- Product/documentation version advances from `v1.19.0` to `v2.0.0` only during successful final documentation.
- Private first-party npm workspace versions remain `0.1.0`.
- No npm publication is part of this release.
- No container-registry publication or cloud deployment is part of this release.
- The release tag is annotated.
- Planned annotation: `PulseGate v2.0.0 - Product/Platform v2`.
- The tag must target the pushed final Sprint 80 documentation commit.
- The tag is created only after post-commit release validation passes.

## Bounded Release Meaning

This milestone does not claim:

- Public SaaS availability.
- Production cloud deployment.
- Production capacity certification.
- SLA or SLO guarantees.
- Enterprise compliance certification.
- Payment or billing readiness.
- Full production-grade multi-tenant isolation.
- npm or container-registry publication.
